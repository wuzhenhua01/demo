# QuickPin - 快速定位标记 App 设计文档

## 1. 产品概述

Android 原生 App，用于一键标记当前位置。极简设计，打开即用，无需注册登录。

- 目标用户：需要快速记录位置的个人用户（如记停车位、标记地点等）
- 核心价值：一键标记，零学习成本

## 2. 技术栈

| 项目 | 选择 |
|------|------|
| 语言 | Kotlin |
| 最低 SDK | API 26 (Android 8.0) |
| 地图 | 高德地图 Android SDK |
| 存储 | Room (SQLite) |
| 架构 | MVVM + 单 Activity + Navigation Component |
| UI | Material 3，浅色模式，主色蓝色 |

## 3. 数据模型

### Pin 实体

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long (自增) | 主键 |
| latitude | Double | 纬度 |
| longitude | Double | 经度 |
| createdAt | Long | 创建时间戳（毫秒） |

极简设计——只存坐标和时间，无备注、分类、照片等附加信息。

## 5. UI 设计

### 5.1 底部导航

两个 Tab：地图 / 列表

### 5.2 地图 Tab

- 全屏高德地图，默认定位到用户当前位置
- 右下角悬浮圆形按钮（FAB），点击即标记当前位置
- 地图上用蓝色圆点标记所有已保存的位置
- 点击标记点弹出底部 Sheet，显示坐标和标记时间，提供「删除」按钮
- 首次打开时请求位置权限

### 5.3 列表 Tab

- 按时间倒序显示所有标记点
- 每行显示：标记序号、坐标（经度, 纬度）、标记时间
- 左滑删除
- 点击某行跳转到地图 Tab 并定位到该标记点
- 空状态显示提示文字「还没有标记，去地图上标记一个吧」

## 6. 核心流程

### 6.1 标记流程

1. 用户打开 App → 请求位置权限（首次）
2. 地图定位到当前位置
3. 用户点击 FAB → 获取当前坐标 → 存入数据库 → 地图上新增标记点 → 显示 Toast「已标记」
4. 如果获取位置失败 → Toast 提示「无法获取位置，请检查定位是否开启」

### 6.2 删除流程

- 地图 Tab：点击标记点 → 底部 Sheet → 点击删除 → 确认对话框 → 删除
- 列表 Tab：左滑 → 点击删除 → 直接删除（无需确认）

## 7. 错误处理

| 场景 | 处理方式 |
|------|----------|
| 位置权限被拒绝 | 显示引导页，说明需要位置权限才能使用 |
| 定位服务未开启 | 提示用户前往设置开启 |
| 获取位置失败 | Toast 提示「无法获取位置，请检查定位是否开启」 |
